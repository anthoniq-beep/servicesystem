为了构建一个符合您公司需求的业务系统，我们需要先明确具体的业务规则。随后我们将进入项目初始化和开发阶段。

### 阶段一：需求与规则确认 (当前阶段)
我们需要您提供以下核心业务规则的详细信息（见下文详细列表）：
1.  **组织架构与权限**：部门层级、角色定义、数据可见性。
2.  **员工管理**：档案字段、入离职流程。
3.  **客户管理 (CRM)**：客户生命周期、公海池规则、跟进记录要求。
4.  **提成结算**：计算公式、结算周期、业绩归属。

### 阶段二：技术选型与初始化
在您确认规则的同时，我们可以先确定技术基础：
1.  **前端**：建议使用 React 或 Vue (如 Ant Design Pro 或 Element Plus) 以快速搭建后台管理界面。
2.  **后端**：建议使用 Node.js (NestJS/Express), Python (Django/FastAPI), 或 Java (Spring Boot)。
3.  **数据库**：建议使用 PostgreSQL 或 MySQL。

### 阶段三：核心模块开发
1.  设计数据库模型 (ER图)。
2.  搭建基础框架 (登录认证、RBAC权限控制)。
3.  按模块开发：组织架构 -> 员工 -> 客户 -> 提成。
