# 业务系统需求规格说明书 (Business System Requirements Specification)

## 1. 组织架构与员工管理 (Organization & Employee Management)

### 1.1 组织架构规则
- **层级结构**: 公司 -> 部门 -> 组 -> 个人
- **角色定义**:
    - 超级管理员 (Admin): 拥有所有权限
    - 部门主管 (Manager): 查看本部门及下属部门数据
    - 销售经理（supervisor）: 查看本人及直属下级销售数据
    - 销售专员/员工 (Employee): 仅查看个人数据
    - 财务 (Finance): 查看提成、合同、回款数据
    - 人事 (HR): 查看员工档案、考勤

### 1.2 员工管理规则
- **状态**: 试用期、正式、离职
- **关键字段**: 姓名、工号、手机号、部门、职位、职等、入职日期

## 2. 客户管理系统 (CRM)

### 2.1 客户状态流转 (Lifecycle)
- 线索(Lead) -> 客资（Chance）-> 约访 (Call) -> 接待 (Touch) -> 考虑（pending）-> 成交 (Deal) -> 流失 (Churned)

### 2.2 公海池规则 (Public Pool Rules)
- **回收机制**: 
    - [30] 多少天未跟进自动掉入公海？ (建议: 15天)
    - [90] 多少天未成交自动掉入公海？ (建议: 90天)
- **领取限制**:
    - [50] 每个销售最多持有多少个客户？ (建议: 50-100个)

### 2.3 查重规则
- 唯一标识: 手机号 / 公司名称 

### 2.4 线索来源（Lead Of From）
- 公司渠道（company）：由部门经理统一分配客资跟进人员
- 个人渠道（individual）： 由销售人员、销售主管或部门经理直接获取的客资，此类客资须由本人填写客资姓名、手机号
    - 根据线索来源方式，需要单独做一个线索来源管理表单，需要实时的管理线索来源及新增线索来源

## 3. 提成结算 (Commission Settlement)

### 3.1 提成规则（Commission Rule）
- 客资（Chance）：公司渠道客资提成比利分配：跟进销售人员[1%]、跟进部门[2%]，个人渠道客资：客资所属销售人员[3%]
- 约访（Call）：成功约访客户至门店的销售人员提成[2%]
- 接待（Touch）：完成接待的销售人员提成[2%]
- 成交（deal）：销售专员[1%]、销售经理[2%]、部门主管[3%]

### 3.2 提成基础 (Base)
- [y] 按回款金额计算 (推荐),回款金额须由部门主管填写，超级管理员复核。

### 3.2 提成比例 (Commission Rate)
- 最终成交的客户按销售流程完成情况提成：
    - 客资（Chance）：公司渠道客资提成比利分配：跟进销售人员[1%]、跟进部门[2%]，个人渠道客资：客资所属销售人员[3%]
    - 约访（Call）：成功约访客户至门店的销售人员提成[2%]
    - 接待（Touch）：完成接待的销售人员提成[2%]
    - 成交（deal）：销售专员[1%]、销售经理[2%]、部门主管[3%]

### 3.3 结算周期
- [Y] 季度结：每年1/4/7/10月发放上季度提成
